# Reference: Terminology Glossary 📖

This glossary defines common cybersecurity terms and concepts referenced throughout the lab manuals.

---

### SUID (Set Owner User ID)
A special file permission in Unix-like systems that allows users to run an executable with the permissions of the file's owner (often `root`), rather than the user running the command.

### CVSS (Common Vulnerability Scoring System)
A standardized scoring system used to assess and communicate the severity and risk level of security vulnerabilities, ranging from `0.0` (Informational) to `10.0` (Critical).

### Loopback Interface
A virtual network interface (typically `127.0.0.1` or `::1`) used by a computer to send network traffic to itself, ensuring the traffic remains completely isolated within the local machine.

### SQL Injection (SQLi)
A vulnerability where an attacker inserts malicious SQL code into input fields, allowing them to manipulate database queries and access unauthorized data.

### Parameterized Query
A secure database coding practice where input values are kept separate from the query structure, preventing SQL injection vulnerabilities.

### File Integrity Monitoring (FIM)
The process of calculating and comparing file hashes (such as SHA256) over time to detect unauthorized changes, modifications, or deletions in sensitive system files.
