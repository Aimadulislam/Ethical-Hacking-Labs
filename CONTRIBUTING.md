# Contributing Guide

Welcome to the Ethical Hacking Labs! We are committed to fostering a safe, constructive, and highly educational repository.

## Contribution Standards
1. **Safety First**: We do NOT accept submissions containing malware payloads, real exploit scripts, reverse shells targeting actual software, phishing frameworks, bypasses, or offensive tools meant to cause harm.
2. **Defensive focus**: Scripts submitted should be audit-focused, monitoring-focused, defensive helpers, or isolated practice environments.
3. **Format & Quality**:
   - Bash scripts must pass `ShellCheck`.
   - Python code must conform to PEP8.
   - Manuals should contain detailed Mermaid diagrams and educational Q&As.

## Submission Workflow
1. Fork the repository.
2. Create a clean feature branch.
3. Add your changes, ensuring no actual API keys, secrets, or vulnerable variables are committed.
4. Open a Pull Request detailing the educational benefit of your submission.
